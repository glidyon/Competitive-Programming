# Text Elements
For me, one of the best approaches for this problem was RLE (Run Length Encoding).
https://en.wikipedia.org/wiki/Run-length_encoding

Basically, for every groups of 0's and 1's, we are making a separate space in our vector array.
For example:

If we have a string which contains a following string of characters:

10110111

We make a loop which will put all of the groups of 1's and 0's in separated cells in the pairs vector.
 ^ZpKvbB84

ch ^rnK2pEAU

am ^ZSRtSq5h

So now we know (ch - character that represents the group, i.e. '1' or '0', am - amount of that character appearing one after
another) that we have 1 one, 1 zero, 2 ones, 1 zero, 3 ones one after each other. We basically
have them in groups and we know how many of them appear after each other now.

So the only last thing we have to do is to check for the best sub-array. We can do that by adding to the sum the amount
of 0's and 1's happening one after another, and if we meet '0' as the 'ch' in our vector, we subtract 1 from the k variable.

We check for the max value in the sub-arrays every time our k is not equal to -1. If it is, it means we have exceeded the limit.
So if we've exceeded our limit of setting the ranges, we basically do another loop which subtracts earlier range. After it hits the
first zero in the previous range, we have our pointer set on the beginning in the current range. And after that loop it checks
the max value and starts checking the next range.

I will use the example from the sample case on atcoder's contest.
n = 14 k = 2
s = 11101010110011

So we will have something like this:
 ^efdtRrRS

At first, we will have a sum which will contain a string like this one:
111, k = 2, sum = 3
We check this sum, which is 3, with our best result which is 0, and obviously we write the 3 to the ans variable.
Next we have 1110, where k = 1 and sum = 4
Notice how we have subtracted 1 from the k, but it is still more than -1, so we do nothing.
We add to the sum until we hit a result like that:
1110101, which is sum = 7, k = 0
And then we have another 0, which makes our k = -1, we go inside the if block.
Notice how we don't write the sum, which would be 8 at this moment, the sub-array would be "11101010" at the moment.

Now, when we are in the loop, we basically subtract first groups in our string and from the sum until we make
our "k" variable equal to 0.
We subtract "111" and "0" group, which makes our new sub-array equal to "1010".
Notice how we move the pointer as we subtract the groups.

Original string: 11101010 110011
Our sub-array: 11101010
Sub-array after the if statement: 1010

Now we have "1010", so k=0, we are adding more to our sub-array until we hit k=-1.
We will have 10101100, then our "k" is equal to -1, and sum is 8, but we don't write it to the ans variable, since it contains
two additional 0's which are exceeding our k, so it's not allowed.

Original string: 11101010 110011
Our sub-array: 10101100
Sub-array after the if statement: 101100

After subtracting from the sum, our sub-array would look like this: 101100, which's sum is equal to 6, and k = 0. We have two ones left.

Our sub-array looks like this: 10101100, and the full string looks like this: 11101010110011
If you've noticed already - we don't have any zeros next, so k will never be equal -1. We add this ones group, and we receive a
sub-array equal to this:
1010110011, which sum's equal to 8. That would be the max among all of the sub-arrays and that's our answer.


Easier example would be the first test case in the example:
n = 5 k = 1
s = 00010

We have a group like this:
'0' '1'  '0'
3   1   1

We add the 3 zeros group to the sum, which is equal to 3 now. Then we check the max, max between 0 and 3 is of course 3, since it
is larger than 0. Notice how our k is equal to 0 now.
Now we have 1, in another iteration of the loop, and we add it. Now our sum is 4. We check the max, now ans = 4. Since 4 > 3.
And then last zero appears, which makes our k = -1, and our sum equal to 5.

We subtract from the sum the group of zeros. It makes our sum equal to 2, and add 1 to k. Which takes us out from the loop.
We also move the "pnt" variable to "point" on the ones group, which is equal to 1. The sum is equal to 2, but 2 < 4 so we don't
write it to the ans variable.


As you can see, we mostly wanted to "flip" the zeros, since they ask us for "the maximum possible number of consecutive
people standing on hands after at most K directions". Obviously, we don't want to flip the ones, since they are the people who already
do the handstand. So we flip the zeros, and to flip that, we had to separate them in groups. After we check every group, we can
switch to the next group. That's why we need to check for the "k" variable. It makes the algorithm select the next subarray and the
maximum possible sum in it.
